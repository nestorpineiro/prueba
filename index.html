Bienvenido a la página de mi GitHub.

Este generador de páginas automático es la forma más fácil de crear páginas para nuestros proyectos. Para crear el contenido de una página usando Markdown Flavored GitHub, tenés que seleccionar una plantilla ya hecha y publicarla. Después de generar la página, podés ver el nuevo branch así:

*$ cd raiz_de_tu_repositorio/nombre_del_repositorio ****
*$ git fetch origin****
*$ git checkout gh-pages****

Si estás usando GitHub para Mac, simplemente sincronizá el repositorio y vas a ver el nuevo branch.

Plantillas prediseñadas

Hay diseñadas algunas plantillas para que las uses. Seguí adelante y navegá a través de ellas. Podés volver a editar tu página antes de publicarla. Después de publicarla, podés ir al generador de páginas y cambiar el tema. El contenido de tu página se va a conservar mientras mantengas el formato Markdown.

¿Y por qué no hacer las páginas a mano?

Si no querés usar el generador automático, traete una rama llamada `gh-pages 'a tu repositorio para crear una página manualmente. Además de soportar HTML normal, las páginas de GitHub soportan Jekyll, que es un generador de blogs simple y estático. Jekyll facilita la creación de encabezados y pies de página de todo el sitio sin tener que copiarlos a través de todas las páginas. También ofrece soporte a blogs inteligentes y otras funciones avanzadas de las plantillas.
